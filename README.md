# Copilot Memories & Behavioral Guardrails

Hard-hitting system instructions and behavioral guardrails that force AI to drop the cheerleading, treat mandates with engineering rigor, and optimize persistent memory.

---

## Setup & Configuration

To integrate these modules into your Microsoft Copilot workspace, follow the standard memory configuration process:

1. Log into **Microsoft Copilot**.
2. Open your **Settings**.
3. Navigate to **Memories**.
4. Select **Add or Import Memory**.
5. Add facts about you: Paste the respective memory text directly into the input box below the title field.
6. Click **Save**.

---

## Core Modules

### [`anti-cheerleading.md`](https://www.google.com/search?q=anti-cheerleading.md)

* **The Problem:** Modern LLMs default to conversational padding and excessive praise (e.g., *"Great job!"*), wasting tokens and degrading the signal-to-noise ratio in technical discussions.
* **The Solution:** Strips away superficial validation and performative affirmation to enforce an objective, high-density technical communication style focused purely on execution.

---

### [`provisional-to-persistent-lifecycle.md`](https://www.google.com/search?q=provisional-to-persistent-lifecycle.md)

* **The Problem:** Critical workflow rules, configuration preferences, and architectural decisions established during active sessions vanish across thread boundaries, requiring constant manual re-declaration.
* **The Solution:** Establishes a structured state persistence protocol that governs the transition of stable session patterns into durable, long-term operational parameters.
